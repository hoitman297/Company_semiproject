Vue.js 전체 흐름 처음부터 끝까지 정리

전체 개요

Vue.js 앱의 전체 구조

Vue 앱은 크게 3가지 영역으로 나뉩니다. 각 영역이 어떤 역할을 하는지 먼저 이해하면 전체 흐름이 쉬워져요.

브라우저→ 프론트엔드 (Vue)→ 백엔드 (Spring)→ DB (MyBatis)
프론트엔드 (Vue.js) ├── index.html ← 브라우저가 처음 읽는 파일 ├── main.js ← Vue 앱 시작점 (딱 1개) ├── App.vue ← 전체를 감싸는 틀 ├── router/ ← URL → 페이지 연결 ├── views/ ← 페이지 파일들 └── components/ ← 재사용 부품들
💡 main.js는 반드시 1개, 나머지는 얼마든지 늘릴 수 있어요.
STEP 1

브라우저가 index.html을 읽는다

주소창에 localhost:5173 입력 → 브라우저가 가장 먼저 index.html을 읽습니다. 이 파일 안에 Vue 앱이 들어갈 빈 자리 #app이 있어요.

<!-- index.html --> <div id="app"></div> ← Vue가 들어올 빈 자리 <script src="/src/main.js"> ← main.js 실행 시작

#app은 규칙이 아니라 관습입니다. #root, #myapp으로 바꿔도 되는데, main.js의 .mount()와 이름만 같으면 됩니다.

💡 index.html은 거의 수정하지 않아요. 탭 제목 바꿀 때 정도만 열어봅니다.
STEP 2

main.js — Vue 앱의 시작점

main.js가 실행되면서 Vue 앱을 만들고 #app 자리에 끼워 넣습니다. 앱 전체에서 쓸 플러그인도 여기서 등록해요.

import { createApp } from 'vue' import App from './App.vue' import router from './router' createApp(App) ← Vue 앱 생성 .use(router) ← 라우터 플러그인 등록 .mount('#app') ← index.html의 #app 자리에 삽입

.use(router)를 해야 앱 전체에서 RouterView, RouterLink 같은 라우터 기능을 쓸 수 있습니다. 등록 안 하면 Vue가 라우터의 존재를 몰라요.

💡 main.js는 얇게 유지 — 등록만 하고 실제 내용은 각 파일에서 만들어요.
STEP 3

App.vue — 모든 페이지를 감싸는 틀

내가 만든 모든 view가 App.vue 안의 RouterView 자리에 들어갑니다. Header, Footer처럼 항상 보이는 공통 부분을 여기에 넣어요.

<template> <Header /> ← 모든 페이지에서 항상 보임 <RouterView /> ← URL마다 바뀌는 내 view가 여기 들어옴 <Footer /> ← 모든 페이지에서 항상 보임 </template>

즉 내가 만든 view는 #app → App.vue → RouterView 순서로 안쪽에 들어가서 화면에 표시됩니다.

STEP 4

router — URL과 페이지를 연결

URL을 보고 어떤 view를 RouterView 자리에 넣을지 결정하는 지도입니다.

// router/index.js import { createRouter, createWebHistory } from 'vue-router' const router = createRouter({ history: createWebHistory(), ← URL 방식 (깔끔한 /home 형태) routes: [ { path: '/', ← 이 URL이면 component: () => import('../views/HomeView.vue') ← 이 파일 보여줘 }, { path: '/about', component: () => import('../views/AboutView.vue') } ] }) export default router ← main.js에서 쓸 수 있게 내보내기
💡 () => import() 방식은 해당 페이지에 실제 접속할 때만 파일을 불러와서 첫 로딩이 빨라져요.
STEP 5

views/ — 페이지 파일

URL 하나에 대응하는 전체 페이지입니다. components/에서 만든 부품을 조립해서 완성된 화면을 만들어요.

<!-- views/HomeView.vue --> <template> <h1>상품 목록</h1> <ProductCard v-for="item in products" :product="item" /> </template> <script setup> import ProductCard from '@/components/ProductCard.vue' import { ref, onMounted } from 'vue' const products = ref([]) ← 반응형 변수 (값 바뀌면 화면 자동 업데이트) onMounted(() => { loadProducts() ← 페이지 열리면 데이터 불러오기 }) </script>
STEP 6

components/ — 재사용 부품

여러 페이지에서 반복해서 쓰는 작은 단위입니다. 한 번 만들면 어디서든 가져다 써요.

<!-- components/ProductCard.vue --> <template> <div class="card"> <h3>{{ product.name }}</h3> ← .value 없이 바로 사용 <p>{{ product.price }}원</p> </div> </template> <script setup> defineProps(['product']) ← 부모 view에서 데이터 받기 </script>
views/ → 완성된 페이지 (레고 완성품) components/ → 재사용 부품 (레고 블록)
STEP 7

ref() 와 .value — 반응형 변수

Vue가 값의 변화를 감지하려면 ref()로 감싸야 합니다. ref()는 값을 객체로 포장해서 변화를 추적해요.

const products = ref([]) // 실제 내부 구조 products = { value: [] ← 진짜 값은 .value 안에 있음 } // JS에서는 .value 필요 products.value = [{ name: '신발' }] ← 화면 자동 업데이트 // 템플릿에서는 .value 생략 {{ products }} ← Vue가 자동으로 .value 벗겨줌 v-for="item in products" ← 마찬가지
💡 .value가 번거로워 보이지만 이 덕분에 화면이 자동으로 업데이트됩니다.
STEP 8

axios — 백엔드에 데이터 요청

Vue는 화면만 담당하고 데이터는 백엔드에서 가져옵니다. axios가 HTTP 요청을 보내는 역할을 해요.

async function loadProducts() { const res = await axios.get('/api/products') // res = 백엔드 응답 전체 (봉투) // res.data = 실제 데이터 (봉투 안 내용물) products.value = res.data // ↑ 이 순간 Vue가 변화 감지 → 화면 자동 업데이트 }
res = { status: 200, ← 성공 여부 data: [ ← 우리가 필요한 실제 데이터 { id: 1, name: '신발', price: 50000 }, { id: 2, name: '가방', price: 80000 }, ] }
STEP 9

Spring — Controller → Service → Mapper

백엔드에서 요청을 받아 처리하는 3단계입니다. Controller가 받고 → Service가 로직 처리 → Mapper가 DB에 SQL 실행해요.

// Controller: URL 요청 받기 @GetMapping("/api/products") public List<Product> getProducts() { return service.getProducts(); } // Service: 비즈니스 로직 public List<Product> getProducts() { return mapper.selectAll(); } // Mapper 인터페이스 List<Product> selectAll();
STEP 10

MyBatis — mapper.xml에서 SQL 실행

MyBatis는 Java 코드 대신 XML 파일에 SQL을 작성해서 DB와 소통합니다. Mapper 인터페이스의 메서드 이름과 XML의 id가 연결돼요.

<!-- mapper.xml --> <mapper namespace="com.example.mapper.ProductMapper"> <select id="selectAll" resultType="Product"> SELECT * FROM products </select> </mapper>
axios 요청→ Controller→ Service→ Mapper→ mapper.xml→ DB→ 화면 업데이트
전체 정리

처음부터 끝까지 한눈에

브라우저 접속 ↓ index.html 읽기 → Vue 앱이 들어갈 #app 자리 준비 ↓ main.js 실행 → createApp().use(router).mount('#app') ↓ App.vue 로드 → 전체 틀 + RouterView 자리 준비 ↓ router 확인 → URL 보고 어떤 view 보여줄지 결정 ↓ views/*.vue 렌더링 → 내가 만든 페이지가 #app 안에 표시 ↓ axios 요청 → 백엔드 API 호출 ↓ Controller → Service → 요청 받아서 로직 처리 ↓ Mapper → mapper.xml → SQL 실행 ↓ DB 조회 → 데이터 반환 ↓ products.value = res.data → 화면 자동 업데이트
💡 Vue는 화면 담당, Spring은 로직 담당, MyBatis는 DB 담당 — 역할이 명확하게 나뉘어요!
